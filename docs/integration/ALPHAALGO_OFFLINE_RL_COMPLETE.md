## 📊 WHAT HAPPENS WHEN YOU ACTIVATE

### Immediate Actions (First Run)

1. **System Initialization**
   - Creates directories: `alphaalgo_autonomous/logs/`, `models/`, `data/`, `reports/`
   - Initializes replay buffer (100k capacity)
   - Sets up training scheduler (24-hour intervals)
   - Configures safety thresholds

2. **Data Collection Begins**
   - Every trade is recorded
   - State-action-reward transitions stored
   - Buffer fills up during live trading
   - Minimum 10k transitions needed before training

### After 24 Hours (First Training Cycle)

3. **Automatic Training**
   - Trains 3 policies in parallel:
     - CQL (Conservative Q-Learning)
     - IQL (Implicit Q-Learning)
     - BCQ (Batch-Constrained Q-Learning)
   - Each trains for 100 epochs
   - Saves models to `alphaalgo_autonomous/models/`

4. **Policy Evaluation**
   - Evaluates each policy with:
     - FQE (Fitted Q Evaluation)
     - DR (Doubly Robust)
     - WIS (Weighted Importance Sampling)
     - CVaR (Conditional Value at Risk)
   - Calculates Sharpe ratio, win rate
   - Ranks policies by risk-adjusted return

5. **Safety Gating**
   - Checks if best policy passes ALL thresholds:
     - FQE ≥ 0.7 ✓
     - DR ≥ 0.65 ✓
     - CVaR ≥ -0.02 ✓
     - Sharpe ≥ 1.5 ✓
     - Win Rate ≥ 55% ✓
   - If passes: Deploy
   - If fails: Keep current policy, log reason

6. **Deployment**
   - Backs up current policy
   - Deploys new policy
   - Starts performance monitoring
   - Logs deployment event

### Continuous Operation

7. **Performance Monitoring**
   - Tracks real-time P&L
   - Monitors Sharpe ratio
   - Checks win rate
   - Measures drawdown
   - Compares to baseline

8. **Auto-Rollback (If Needed)**
   - Detects performance degradation
   - Reverts to previous policy
   - Logs rollback event
   - Alerts operator
   - Continues monitoring

9. **Next Training Cycle**
   - Repeats every 24 hours
   - Uses updated data
   - Trains new candidates
   - Continuous improvement
